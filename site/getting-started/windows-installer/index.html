Windows Installer¶

The Windows installer sets up Bambuddy natively on Windows without Docker.

It checks the required tools, clones Bambuddy, creates a Python virtual environment, installs dependencies, creates a start script, adds optional firewall access, and can register Bambuddy as a Windows Service.

Quick Start¶

Run PowerShell as Administrator and start the installer:

powershell -ExecutionPolicy Bypass -Command "iwr -useb https://raw.githubusercontent.com/maziggy/bambuddy/main/install/windows-installer.ps1 -OutFile windows-installer.ps1; .\windows-installer.ps1"

Recommended

Use the default installation path unless you have a specific reason to change it.

Default Installation Path¶

By default, Bambuddy is installed to:

C:\Bambuddy

This path avoids common permission issues with C:\Program Files.

Administrator Rights¶

The installer relaunches itself through UAC if it is not elevated.

  • Installs Git or Python with winget
  • Creates firewall rules
  • Adjusts folder permissions
  • Registers Bambuddy as a Windows Service

Installation Flow¶

git clone https://github.com/maziggy/bambuddy.git
cd bambuddy
python -m venv venv
.\venv\Scripts\python.exe -m pip install --upgrade pip
.\venv\Scripts\pip.exe install -r requirements.txt
.\venv\Scripts\python.exe -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000

Port Selection¶

The default port is 8000. The installer lets you choose a different port during setup.

Logging¶

FileDescription
install.logInstaller actions and setup errors
bambuddy-runtime.logRuntime output
bambuddy-runtime-error.logRuntime errors

Windows Service¶

The installer can register Bambuddy as a Windows Service named Bambuddy.

The service is created through NSSM so Bambuddy can start automatically after reboot and run without an open PowerShell window.

Get-Service Bambuddy
Start-Service Bambuddy
Stop-Service Bambuddy
Restart-Service Bambuddy
sc.exe qc Bambuddy

Troubleshooting¶

Service does not start¶

First test the start script manually:

powershell.exe -ExecutionPolicy Bypass -File "C:\Bambuddy\Start-Bambuddy.ps1"

Then check the logs in C:\Bambuddy.

Delete and recreate the service¶

Stop-Service Bambuddy -Force -ErrorAction SilentlyContinue
C:\Bambuddy\nssm\nssm.exe remove Bambuddy confirm

Next Steps¶
